                     To initialize git we have to use one command i.e git init.
                  

                     There are 3 stages in gitFlow.They are:
                     Working directory  --git init 
                     staging area    -- git add
                     local and remote repository -- git remote add origin, git branch -M main and git push origin main.
                  

                    The command used to stage the code is git add .
                  

                    The command used to commiting for staged changes are git commit -m "message" --The message should be meaningful message.
                  

                     The command used to communicate from local repo to remote repo is git remote add origin 
                  

                    Repository is nothing but  A storage location for all project files, which tracks changes.
                    It can be 2 types -- Public: Accessible to everyone.
                                         Private: Accessible only to the owner or authorized users.
                    It is also known as “Storage Chunk.”
                  

                     There is a one way to push code from local to remote repo is --Git push origin main.
                  

                    To creat a branch we can use the command -- git branch branchname.
                    If you want to create a new branch and want to switch to other branch in one command we have to use -- git checkout -b branchname.
                  

                    To check for the branch name we have to use the command -- git branch.
                  

                    To switch the branch from current branch to another branch we have to use git checkout branchname.
                  

                     The git log --oneline command is used to check the commit history.
                  

                     The git status command is used to display the state of working directory and staging area.
                     It shows which changes have been stages and which have not and which files are not being tracked by git.
                     The git status command is important for understanding the current status of your project.
                  

                    In 3 ways we can get the code form remote to local.
                    git clone -- Git clone is nothing but downloading/copying the existing file from the remote repo to local.
                    git fetch -- Git fetch command is used to downloads the updates from the remote repo but it doesn't combine with the current branch. 
                    git pull -- The git pull command is used to fetch and merge code changes from remote repo to local repo.
